# 演策项目需求沉淀

## 用户选择题回答

### 1. 改进方向
- **全部都做**（除了本地降级，用户强调要测试真实的LLM效果）
- 优先：深化追问、动态生成Agent、优化总结质量

### 2. 视觉效果
- 修复环绕符号大小问题（改回之前的大小）
- 在当前基础上加入更多可互动的特效
- 不同流程阶段设计不同的特效

### 3. 演的角色定位
- **全局都有**，演是一个多Agent团队的总称
- 要按当前最先进、最具有生产级的multiAgent框架来构造
- Agent设计要真正体现Agent特性

---

## 核心改进计划

### 一、MultiAgent架构重构
1. **动态Agent生成**：演根据用户问题动态创建专属Agent
2. **Agent团队协作**：多个Agent之间可以协作、辩论、互相参考
3. **状态管理**：设计完善的状态机控制Agent间的流转
4. **记忆机制**：Agent具有短期记忆，能记住对话历史

### 二、追问深度增强
1. 每个Agent最多追问3层
2. Agent根据回答质量决定是否继续追问
3. 追问问题要层层递进，逐步深入

### 三、总结质量优化
1. 演整合所有Agent的观点
2. 生成更有针对性的选项
3. 每个选项包含关键点摘要和卦象推荐

### 四、特效改进
1. 修复卦象环绕符号大小
2. 为每个流程阶段设计独特的互动特效
3. 增加用户可交互的元素

---

## 技术实现要点

### MultiAgent架构
- 使用状态机管理流程阶段
- Agent间通过共享上下文进行协作
- 支持动态创建和销毁Agent
- 实现Agent间的消息传递机制

### 追问逻辑
- 每个Agent独立判断是否继续追问
- 追问深度可配置（默认3层）
- 回答质量评估机制

### 总结机制
- 收集所有对话历史
- 提取关键信息点
- 生成结构化选项
- 结合周易卜卦进行推荐
