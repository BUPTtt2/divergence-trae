## 4. 核心API接口

### Agent编排

```
POST /api/agent/analyze
  Body: { question: string }
  Res:  { agents: [{id,name,stance,persona,color,glow,symbol}], reasoning: string }

POST /api/agent/dialogue
  Body: { agentId, agentPersona, question, previousDialogues? }
  Res:  SSE stream → 逐字返回对话内容
```

### 周易起卦

```
POST /api/divination/cast
  Body: { question: string }
  Res:  { hexagram: {name,symbol,element}, changingLines: [], 
          originalText: 卦辞/爻辞, interpretation: LLM解读 }

POST /api/divination/interpret
  Body: { hexagram, question, agentDialogues }
  Res:  { verse, framework, powerfulQuestion, summary }
```

### 天命牌

```
GET  /api/cards?userId=xxx          # 获取卡牌列表
POST /api/cards                     # 保存天命牌
PUT  /api/cards/:id                 # 编辑天命牌
DELETE /api/cards/:id               # 删除
POST /api/cards/:id/share           # 分享到社区
```

### 社区

```
GET  /api/community/posts?sort=hot|new&tag=xxx
POST /api/community/posts           # 发帖
POST /api/community/posts/:id/reply # 回复
POST /api/community/posts/:id/like  # 点赞
```

### 其他

```
GET  /api/daily?date=YYYY-MM-DD     # 每日运势
GET  /api/calendar?userId=xxx       # 推演日历
GET  /api/achievements?userId=xxx   # 成就
```

## 5. 关键数据链路

### 推演主链路（最核心）

```
1. 用户输入问题
2. 前端 → POST /api/agent/analyze
   后端: 智谱LLM分析问题 → 返回Agent列表（1-6个动态）
3. 前端接收Agent列表 → 3D场景渲染Agent位置
4. 前端逐个 → POST /api/agent/dialogue (SSE)
   后端: 对每个Agent调LLM → 流式返回对话
5. 前端: 对话在3D场景打字机效果浮现
6. 所有Agent发言完毕 → 前端 → POST /api/divination/cast
   后端: 金钱卦算法 → 返回卦象+卦辞
7. 前端 → POST /api/divination/interpret
   后端: LLM综合卦象+对话 → 返回签/反问/框架
8. 前端: 用户选择路径 → 生成天命牌
9. 前端 → POST /api/cards → 保存到后端
10. 用户可选 → POST /api/cards/:id/share → 发到社区
```

### 异常降级链路

```
正常: 智谱glm-4-flash
  ↓ 超时8s / 报错
降级1: 魔搭
  ↓ 超时8s / 报错
降级2: DeepSeek
  ↓ 超时8s / 报错
降级3: 本地预设（inferenceEngine.js现有的SMART_PRESETS）
  → 全程不白屏，用户体验完整
```

## 6. 数据库Schema

```sql
-- 天命牌
CREATE TABLE cards (
  id          TEXT PRIMARY KEY,
  user_id     TEXT NOT NULL,
  gua         TEXT,
  trigram     TEXT,
  element     TEXT,
  title       TEXT,
  question    TEXT,
  decision    TEXT,
  verse       TEXT,
  summary     TEXT,
  advisors    JSONB,
  pillars     JSONB,
  question_type TEXT,
  powerful_question TEXT,
  framework   TEXT,
  is_shared   BOOLEAN DEFAULT FALSE,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 社区帖子
CREATE TABLE posts (
  id          TEXT PRIMARY KEY,
  user_id     TEXT NOT NULL,
  user_name   TEXT,
  card_id     TEXT REFERENCES cards(id),
  title       TEXT,
  content     TEXT,
  tag         TEXT,
  likes_count INT DEFAULT 0,
  replies_count INT DEFAULT 0,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 回复
CREATE TABLE replies (
  id          TEXT PRIMARY KEY,
  post_id     TEXT REFERENCES posts(id),
  user_id     TEXT NOT NULL,
  user_name   TEXT,
  content     TEXT,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 点赞
CREATE TABLE likes (
  user_id     TEXT,
  post_id     TEXT REFERENCES posts(id),
  created_at  TIMESTAMP DEFAULT NOW(),
  PRIMARY KEY (user_id, post_id)
);

-- 推演记录
CREATE TABLE divination_records (
  id          TEXT PRIMARY KEY,
  user_id     TEXT NOT NULL,
  question    TEXT,
  gua         TEXT,
  agent_ids   JSONB,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 每日签到
CREATE TABLE daily_checkins (
  user_id     TEXT,
  date        DATE,
  gua         TEXT,
  fortune     JSONB,
  PRIMARY KEY (user_id, date)
);
```

## 8. 权限、隔离和边界

| 边界 | 规则 |
|------|------|
| API Key | 只在后端 `.env`，前端永远不可见 |
| CORS | 后端只允许 surge.sh 域名 + localhost |
| 限流 | 每IP每分钟最多10次推演，30次API调用 |
| 用户身份 | 前端生成匿名UUID存localStorage，后端不认证只记录 |
| Agent数量 | 后端硬限制1-6个，防止LLM返回异常 |
| LLM超时 | 8s超时自动降级到下一提供商 |
| 输入过滤 | 问题文本做XSS过滤，限500字 |
| 数据隔离 | 用户只能编辑/删除自己的卡牌和帖子 |
