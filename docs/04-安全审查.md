# 安全审查报告

> 审查时间: 2026-07-08
> 审查范围: `src/` 全目录(75 个文件)
> 应用类型: 纯前端 React + Three.js 应用(演策 AI 决策推演)

## 审查结果总览

该应用为纯前端应用,无后端 API,安全风险整体偏低。

| 优先级 | 问题数 | 状态 |
|--------|--------|------|
| 严重 | 0 | 无 |
| 高 | 0 | 无 |
| 中 | 1 | textarea 无长度限制 |
| 低 | 2 | VITE_LLM_API_KEY 潜在泄露, 缺少 CSP |
| 已修复 | 2 | textarea maxLength, ErrorBoundary |

## 详细问题

### 问题1: [中] textarea 无长度限制

- 入口: Game.jsx 推演输入
- 模块: `src/pages/Game.jsx:538`
- 行为: 用户可输入超长文本(数万字符),导致 setState 性能问题、localStorage 配额溢出
- 修复: 已添加 `maxLength={500}` + `slice(0, 500)` 双重限制
- 状态: 已修复

### 问题2: [低] VITE_LLM_API_KEY 潜在泄露

- 入口: LLM 推理引擎
- 模块: `src/services/inferenceEngine.js:99`
- 行为: `VITE_` 前缀环境变量会被打包进客户端 bundle,若未来启用 LLM 调用,API key 会泄露到前端
- 修复: 当前 `callLlmPipeline` 直接 throw,apiKey 未被使用。未来启用时应改为后端代理
- 状态: 潜在风险,当前不触发

### 问题3: [低] 缺少 CSP 策略

- 入口: 全局
- 模块: `index.html` / `vite.config.js`
- 行为: 无 Content-Security-Policy meta 标签或 HTTP header
- 修复: 上线前在 index.html 添加 CSP meta 或在服务器配置 CSP header
- 状态: 建议修复

## 无问题项

- localStorage: 3 处使用,均包裹 try-catch,无明文敏感数据
- XSS: 0 处 dangerouslySetInnerHTML / eval / innerHTML,React 默认转义
- 硬编码密钥: 无,API key 通过环境变量注入
- 第三方依赖: 版本均新(React 19, Three 0.185, Vite 8),无已知漏洞
- 外部资源: 所有外部加载均为 HTTPS(Google Fonts, trae-api-cn 图片)
- sessionStorage/cookie/postMessage: 未使用

## 结论

该纯前端应用安全状况良好。已修复唯一的实际问题(textarea 长度限制)。VITE_LLM_API_KEY 为潜在风险,当前不触发,留作已知限制。
