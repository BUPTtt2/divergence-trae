# 演策 · 八卦推演引擎 — 交付文档

## 1. 项目概述

演策（YAN CE）是一款基于八卦推演的决策辅助应用，融合中国传统文化与现代 AI 技术，帮助用户通过多角度分析做出更明智的决策。

**核心功能：**
- 推演台：多 Agent 辩论决策系统
- 命签卡：命运卡牌生成与收藏
- 演对话：AI 助手持续对话
- 社区：用户分享与智囊市集
- 每日卦签：每日运势

## 5. 认证系统设计

### 5.1 JWT 双令牌机制

| Token 类型 | 有效期 | 用途 |
|------------|--------|------|
| Access Token | 15 分钟 | API 请求鉴权 |
| Refresh Token | 30 天 | 刷新 Access Token |

### 5.2 认证流程

```
前端初始化
    ↓
检查 localStorage 是否有 token
    ↓
┌── 有有效 token ──→ 直接使用
└── 无 token 或过期 ──→ 尝试刷新
    ↓
┌── 刷新成功 ──→ 更新 token，继续使用
└── 刷新失败 ──→ 匿名登录
    ↓
┌── 后端可达 ──→ 创建云端匿名用户，返回 token
└── 后端不可达 ──→ 降级本地模式，生成本地匿名 ID
```

### 5.3 用户状态

| 状态 | 说明 | 数据存储 |
|------|------|----------|
| `loading` | 初始化中 | - |
| `guest` | 完全未登录 | - |
| `anonymous` | 匿名云端用户 | Cloudflare D1 |
| `registered` | 已注册用户 | Cloudflare D1 |
| `offline` | 离线降级模式 | localStorage |

## 7. 部署配置

### 7.1 前端部署（Cloudflare Pages）

**环境变量：**
```bash
VITE_API_BASE=https://yance-bagua-engine.workers.dev
```

**部署命令：**
```bash
npm run build
# 直接上传 dist 目录到 Cloudflare Pages
```

### 7.2 后端部署（Cloudflare Workers）

**前置条件：**
1. 安装 wrangler CLI：`npm install -g wrangler`
2. 登录：`wrangler login`

**配置步骤：**

1. 创建 D1 数据库：
```bash
wrangler d1 create yance-bagua
# 记录 database_id
```

2. 创建 KV 命名空间：
```bash
wrangler kv:namespace create SESSIONS
# 记录 id
```

3. 设置环境变量：
```bash
wrangler secret put JWT_SECRET
# 输入一个安全的随机字符串

wrangler secret put LLM_API_KEY
# 输入魔搭 API Key
```

4. 执行数据库迁移：
```bash
wrangler d1 execute yance --remote --file=./migrations/0001_init.sql
```

5. 部署：
```bash
wrangler deploy
```

## 10. 关键设计决策

### 10.1 离线降级策略

- 后端不可达时自动降级到 localStorage 存储
- 匿名用户首次访问优先尝试云端注册，失败则本地生成 ID
- 数据同步：上线后自动迁移本地数据到云端

### 10.2 安全设计

- JWT 使用 HS256 签名
- 密码使用 Argon2 哈希
- refresh token 存储哈希值，防止泄露
- 所有输入使用 Zod 校验
- CORS 白名单限制
- KV 限流：每用户每分钟 60 次请求

### 10.3 性能优化

- 前端使用 Vite 构建，代码分割
- 后端使用 KV 缓存热点数据
- SSE 流式输出减少等待时间
- 数据库索引优化

## 11. 注意事项

1. **Cloudflare 免费额度**：Workers 每天 100,000 次请求，D1 免费存储有限
2. **LLM 成本**：需监控 API 调用量，避免超支
3. **JWT_SECRET**：务必使用安全的随机字符串，定期轮换
4. **CORS_ORIGIN**：生产环境只允许信任的域名
5. **数据库备份**：定期导出 D1 数据，防止数据丢失
