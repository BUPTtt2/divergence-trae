# 推理局界面设计文档

## 1. 概述

本设计文档描述了推理局（推演局）界面的重新设计方案，目标是创建一个具有中国风水墨风格、纪念碑谷极简几何美学的沉浸式决策推演界面。

### 1.1 设计目标

- **视觉风格**：中国风水墨 + 八卦推演 + 纪念碑谷极简几何
- **交互体验**：缓慢、空灵、仪式感强的沉浸节奏
- **性能要求**：Web APP 友好，快速渲染，支持多用户并发
- **扩展性**：预设脚本树实现，预留 LLM 接口

### 1.2 核心场景

用户输入决策问题 → 光球分析 → Agent 依次浮现对话 → 光球总结 → 分支选择 → 用户移动到目标方块

- **卦象符文**：所有方块表面都有八卦卦象或传统符号
- **漂浮空灵**：缓慢浮动的节奏，水墨晕染的朦胧感

### 2.2 背景设计

## 3. 核心组件设计

### 3.1 用户光球

- **形态**：多层嵌套的金色光球，最外层是半透明光膜，中间是旋转的太极图，核心是发光球体
- **材质**：哑光金色 + 发光边缘
- **动效**：
  - 缓慢旋转，底部有淡淡的光晕
  - 输入问题时上方浮现文字气泡（打字机效果）
  - 响应时脉动发光

### 3.2 光球（演）

- **形态**：多层嵌套的球体，最外层是半透明光膜，中间是旋转的八卦图，核心是发光金色球体
- **动效**：
  - 外层光膜缓慢脉动（呼吸感）
  - 八卦图持续缓慢旋转
  - 分析阶段：周围有旋转的卦象粒子
  - 总结阶段：所有卦象粒子汇聚到光球周围

### 3.3 Agent 形态设计

#### 钱谷（财务视角 · 金色）

- **形态**：古钱币——外圆内方，边缘有弧度，中间有方孔
- **技术实现**：`LatheGeometry` + 自定义 Shader 实现金属光泽和边缘发光
- **动效**：缓慢自转，边缘有金色粒子环绕，活跃时金属光泽更亮

#### 路向（职业视角 · 绿色）

- **形态**：罗盘——圆形底座，上面有指针和八卦刻度
- **技术实现**：`CylinderGeometry` + 自定义箭头 `BufferGeometry`
- **动效**：指针缓慢摆动，刻度上的卦象符号逐个亮起

#### 风眼（风险视角 · 红色）

- **形态**：风暴眼——螺旋状几何体，中心是空洞，边缘是旋转叶片
- **技术实现**：自定义 `BufferGeometry` + Shader 实现漩涡纹理
- **动效**：整体持续旋转，红色光芒在空洞中闪烁

#### 心禾（情感视角 · 粉色）

- **形态**：莲花——由几片优雅花瓣组成，中心是花蕊
- **技术实现**：`Shape` + `ExtrudeBufferGeometry` + Shader 实现柔和渐变
- **动效**：花瓣缓慢开合（呼吸感），周围有粉色粒子飘散

#### 镜渊（反思视角 · 紫色）

- **形态**：水波纹菱形——钻石切割般的菱形，表面有水波纹纹理
- **技术实现**：自定义 `BufferGeometry` + Shader 实现镜面反射
- **动效**：菱形缓慢翻转，表面水波纹持续流动

#### 云图（宏观视角 · 蓝色）

- **形态**：云朵星盘——蓬松的云朵形状，表面有星象纹理
- **技术实现**：多个 `SphereGeometry` 组合 + Shader 动态生成星象
- **动效**：云朵缓慢飘动变形，星星逐个闪烁

#### 震行（行动视角 · 橙红色）

- **形态**：雷电视觉——棱角分明的多面体，表面有闪电纹路
- **技术实现**：`OctahedronGeometry` 变形 + Shader 实现闪电纹路和能量脉冲
- **动效**：棱角缓慢旋转，闪电纹路间歇性闪烁，活跃时能量脉冲增强

#### 兑言（交流视角 · 青碧色）

- **形态**：水波涟漪——多层同心圆环，像水面泛起的涟漪
- **技术实现**：多个 `TorusGeometry` 同心叠加 + Shader 实现水波流动
- **动效**：圆环由内向外缓慢扩散，像声波传播，活跃时扩散速度加快

### 3.4 选择方块

- **形态**：比普通方块更大，表面更简洁，只有卦象和文字标识
- **动效**：从虚空中一个个升起，悬停时放大发光，选中时发出强烈光芒

### 3.5 阶梯

- **形态**：由多个小方块组成的阶梯，等距三面着色
- **动效**：从起点方块开始依次升起，像建造道路

### 3.6 对话浮层面板

- **位置**：右侧边缘，覆盖约 30% 宽度
- **样式**：无边框，水墨线分隔，半透明宣纸质感背景
- **内容**：当前对话气泡、完整对话记录、输入框

## 4. 交互流程与阶段划分

### 4.1 阶段一：立卦输入

- **场景**：虚空中只有用户光球和初始方块平台
- **动效**：光球缓慢旋转，背景八卦纹理缓慢转动
- **交互**：用户输入决策问题，光球上方浮现文字气泡

### 4.2 阶段二：光球分析（3-5秒）

- **动效**：
  - 光球外层光膜脉动加快
  - 周围出现旋转的卦象粒子（6个卦象围绕光球旋转）
  - 粒子颜色逐渐变化，从淡灰变成各自 Agent 的颜色
  - 背景八卦纹理旋转加速

### 4.3 阶段三：Agent 浮现

- **动效**：
  - 对应 Agent 依次从虚空中升起（0.2-0.3秒间隔）
  - 升起时伴随水墨晕染效果
  - 在八卦位置环绕光球排列
  - 卦象从模糊逐渐变清晰

### 4.4 阶段四：Agent 对话

- **动效**：
  - 当前活跃的 Agent 微微放大，卦象变亮，底部光晕增强
  - 文字气泡从 Agent 上方飘出（打字机效果）
  - 其他 Agent 保持悬浮但亮度降低
  - 用户光球响应时脉动发光

### 4.5 阶段五：光球总结

- **动效**：
  - 所有 Agent 的卦象粒子汇聚到光球周围
  - 光球发出强烈金色光芒
  - 八卦图快速旋转
  - Agent 方块缓缓退到背景（变暗、变小）

### 4.6 阶段六：分支选择

- **动效**：选择方块从虚空中一个个升起，悬停时放大发光
- **交互**：用户点击选择方块，生成阶梯，光球沿阶梯飘动

### 5.2 决策问题类型与 Agent 召唤规则

光球（演）作为主控，根据问题类型召唤 Agent：

```javascript
// 问题类型 → 召唤的动态 Agent（常驻 Agent 自动加入）
const QUESTION_TYPES = {
  career:       { agents: ['qiangu', 'luxiang', 'yuntu', 'zhenxing'] },
  finance:      { agents: ['qiangu', 'yuntu'] },
  relationship: { agents: ['xinhe', 'duiyan'] },
  life:         { agents: ['xinhe', 'luxiang', 'yuntu'] },
  action:       { agents: ['zhenxing', 'luxiang'] },
  communication:{ agents: ['duiyan', 'xinhe'] },
};

// 获取本次决策的完整 Agent 列表
function getAgentsForQuestion(questionType) {
  const permanent = AGENTS.filter(a => a.role === 'permanent');
  const dynamic = QUESTION_TYPES[questionType]?.agents
    .map(id => AGENTS.find(a => a.id === id))
    .filter(Boolean) || [];
  return [...permanent, ...dynamic];
}
```

## 6. 技术实现方案

### 6.1 技术栈

- **3D 渲染**：Three.js + @react-three/fiber + @react-three/drei
- **动画**：Framer Motion（2D）+ useFrame（3D）
- **状态管理**：React Context
- **样式**：Tailwind CSS + 自定义 Shader

### 6.2 组件结构

```
GameBoard/
├── Board3D.jsx          # 3D场景容器
│   ├── GradientBackground.jsx  # 水墨背景
│   ├── UserOrb.jsx      # 用户光球
│   ├── LightOrb.jsx     # 推演光球
│   ├── AgentsLayer.jsx  # Agent层（6种形态）
│   ├── ChoiceBlocks.jsx # 选择方块
│   ├── PathsLayer.jsx   # 阶梯路径
│   └── DecorationsLayer.jsx   # 装饰粒子
└── ChatOverlay.jsx      # 对话浮层面板
```

### 6.3 关键技术点

- **水墨效果**：使用自定义 Shader 实现水墨晕染和纹理
- **卦象渲染**：使用 Canvas 生成卦象纹理，贴到几何体表面
- **性能优化**：使用 InstancedMesh 渲染重复元素，合并几何体

### 6.4 水墨效果实现

**水墨晕染 Shader**：
- 使用噪声函数模拟水墨扩散效果
- 边缘模糊处理，模拟毛笔笔触
- 透明度渐变，从中心向外逐渐变浅
- 颜色：深灰色到浅灰色的渐变

**水墨山峦背景**：
- 使用多层透明平面叠加
- 每层平面上有模糊的山峦剪影
- 不同层有不同的透明度和移动速度
- 模拟深度感

### 6.5 卦象纹理生成

**Canvas 生成流程**：
1. 创建 512x512 Canvas
2. 设置书法字体（如"Ma Shan Zheng"或"ZCOOL XiaoWei"）
3. 绘制卦象符号（使用 Unicode 字符或自定义路径）
4. 添加纹理效果（宣纸纹理叠加）
5. 导出为 PNG 纹理，贴到几何体表面

**卦象符号**：
- 乾 ☰、坤 ☷、震 ☳、巽 ☴、坎 ☵、离 ☲、艮 ☶、兑 ☱
- 太极图：使用 Canvas 绘制黑白阴阳鱼

### 6.6 对话浮层面板细节

**布局**：
- 宽度：320px（约占屏幕 25-30%）
- 位置：右侧边缘，固定定位
- 高度：全屏

**样式**：
- 背景：半透明宣纸质感（`rgba(250,248,240,0.9)`）
- 边框：左侧一条细细的墨线（`#808080`，1px）
- 圆角：无圆角，保持简洁

**内容区域**：
1. **顶部标题**："推演对话"，书法字体，居中
2. **对话记录**：可滚动区域，显示所有对话
3. **当前对话气泡**：高亮显示当前正在进行的对话
4. **输入框**：底部输入区域，用户回答 Agent 问题时使用

**交互**：
- 悬停在 Agent 名字上显示详细信息
- 点击对话气泡可展开/收起
- 输入框支持回车键发送
