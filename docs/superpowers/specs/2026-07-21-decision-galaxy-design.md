# 演策 - 决策星系系统设计文档

## 一、产品定位

**沉浸式AI决策推演游戏**

融合传统文化与AI技术，打造一个让用户产生"哇塞"时刻的决策辅助产品。

## 二、核心创意

### 2.1 决策星系系统

**概念**：将用户的决策体系可视化成一个3D星系，每个决策领域是一个独立的星球。

**设计方案：**

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    │           🌌 决策星系               │
                    │                                     │
        ┌───────────│───────────┐              ┌─────────│─────────┐
        │           │           │              │         │         │
        ▼           ▼           ▼              ▼         ▼         ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │  职场星  │ │  生活星  │ │  情感星  │   │  财富星  │ │  健康星  │ │  学习星  │
   │   球     │ │   球     │ │   球     │   │   球     │ │   球     │ │   球     │
   └─────────┘ └─────────┘ └─────────┘   └─────────┘ └─────────┘ └─────────┘
        │           │           │              │         │         │
        └───────────┴───────────┘              └─────────┴─────────┘
                 前排（高频）                          后排（低频）
```

**功能设计：**

1. **星系导航**
   - 用户可以在3D空间中旋转、缩放、平移视角
   - 点击某个星球进入该决策领域
   - 星球大小反映该领域的推演次数
   - 星球亮度反映该领域的活跃度

2. **星球内部**
   - 该领域的所有推演记录
   - 相关的Agent团队（常驻军师）
   - 专属的卦象解读
   - 领域成就徽章

3. **星际跃迁**
   - 将推演从一个星球跃迁到另一个星球
   - 对比同一问题在不同领域的推演结果
   - 跨领域的决策综合分析

4. **星系成长**
   - 用户使用某个领域越多，该星球越大越亮
   - 解锁新的星球（初始只有3个，随着使用解锁）
   - 星系中心是"演"的核心，连接所有星球

### 2.2 推演剧本创作

**概念**：用户可以创建自己的决策场景剧本，分享给其他人体验。

**功能设计：**
- 用户定义决策主题
- 用户预设Agent角色和立场
- 用户设定推演流程和关键节点
- 生成分享链接

### 2.3 Agent养成系统

**概念**：用户可以培养自己的专属Agent，让它成长和进化。

**功能设计：**
- 自定义Agent获得经验值
- 升级后解锁新功能和外观
- 训练Agent的对话风格和专业领域

### 2.4 Agent Marketplace

**概念**：用户可以创建、分享、交易自定义Agent。

**功能设计：**
- 发布自定义Agent到市场
- 其他用户可以领养
- 创作者获得积分或奖励
- 热门Agent排行榜

### 2.5 决策时间胶囊

**概念**：用户可以封存自己的决策，未来再打开回顾。

**功能设计：**
- 封存决策到时间胶囊
- 设置开启时间
- 到时提醒用户回顾
- 对比当时想法和实际结果

## 三、技术架构

### 3.1 前端架构

```
┌─────────────────────────────────────┐
│           前端展示层               │
│  Three.js 3D场景 + React UI       │
├─────────────────────────────────────┤
│           游戏逻辑层               │
│  状态机管理 + 事件系统 + 成就系统  │
├─────────────────────────────────────┤
│           AI服务层                 │
│  LLM调用 + Agent框架 + 内容生成   │
├─────────────────────────────────────┤
│           数据持久层               │
│  用户数据 + 推演记录 + 收藏系统   │
└─────────────────────────────────────┘
```

### 3.2 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 3D渲染 | Three.js | 决策星系、卦象展示 |
| 动画 | Framer Motion | UI动画、过渡效果 |
| 状态管理 | React Context | 全局状态、游戏状态 |
| AI调用 | API Client | LLM接口调用 |
| 数据存储 | LocalStorage | 用户数据持久化 |

## 六、设计原则

1. **沉浸式体验**：3D场景带来视觉震撼
2. **AI价值**：多Agent真正发挥作用，提供有价值的建议
3. **游戏化设计**：成就、收集、社交等游戏元素
4. **传统文化**：周易卦象、水墨风格
5. **用户创作**：开放创作空间，让用户有施展空间
