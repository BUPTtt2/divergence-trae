# 演策项目上线路线图

## 核心目标
1. **全流程走通** - 从提问到得出结论，每个环节都能正常运行
2. **演做核心判断** - 演负责分析问题、规划流程、生成总结
3. **Agent多选问答** - Agent一次问多个问题，用户选择回答，增加互动乐趣
4. **流程有价值** - 每个环节都有实际意义，不是走过场

## 改进计划

### 第一阶段：核心功能改进

#### 1.1 Agent多选问答
- Agent一次生成3-5个问题
- 用户以选择题形式回答
- 每个问题可以设置权重
- 根据用户选择决定是否继续追问

#### 1.2 演做核心判断
- 演分析问题后决定Agent阵容
- 演根据对话进展调整流程
- 演在关键节点做决策
- 演生成最终总结和选项

#### 1.3 流程优化
- 减少等待时间
- 增加反馈机制
- 优化状态流转

### 第二阶段：体验增强

#### 2.1 互动特效
- 每个阶段独特的视觉效果
- 用户操作有即时反馈
- 卦象互动展示

#### 2.2 性能优化
- LLM调用缓存
- 请求合并
- 加载状态优化

### 第三阶段：测试上线

#### 3.1 全流程测试
- 测试各种问题类型
- 验证边界情况
- 确保异常处理

#### 3.2 部署优化
- 配置优化
- 域名绑定
- 监控设置

## 实现细节

### Agent多选问答实现

**后端接口修改** (`/api/agents/ask`)：
- 新增 `multiQuestions` 参数
- 返回多个问题选项

**前端组件修改** (`AgentDialogueOverlay`)：
- 支持展示多个问题
- 选择题形式回答
- 进度条显示

### 演做核心判断实现

**前端逻辑修改** (`Game.jsx`)：
- 演分析问题后决定Agent
- 演根据回答质量判断是否继续
- 演生成总结和选项

### 测试用例

1. **职业选择**：用户问"该不该跳槽"
   - 演分析问题类型
   - 召唤钱谷、路向、风眼等Agent
   - 每个Agent问2-3个问题
   - 用户选择回答
   - 演生成总结和选项

2. **情感问题**：用户问"该不该表白"
   - 演分析问题类型
   - 召唤心禾、镜渊、兑言等Agent
   - 每个Agent问2-3个问题
   - 用户选择回答
   - 演生成总结和选项

3. **投资决策**：用户问"该不该买股票"
   - 演分析问题类型
   - 召唤钱谷、风眼、云图等Agent
   - 每个Agent问2-3个问题
   - 用户选择回答
   - 演生成总结和选项
