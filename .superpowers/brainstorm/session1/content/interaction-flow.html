迷雾揭开机制 · 三种交互节奏

决定每个节点的体验：到节点后发生什么、怎么揭开迷雾、Agent 扮演什么角色

A · 讨论 → 揭雾 → 选择

到节点 → Agent 依次发言分析 → 讨论结束自动揭开下一层路径 → 玩家选一条继续。节奏流畅，玩家主要是听和选。

推荐 · 符合你说的"讨论分析后再揭开迷雾"

B · 问答对话 → 揭雾 → 选择

到节点 → Agent 逐个向玩家提问，玩家回答 → 答完几轮后揭开路径。玩家参与感强，但交互成本高，节奏慢。

适合深度推演，不适合快速 demo

C · 旁观 → 手动揭雾 → 选择

到节点 → 看 Agent 讨论（随时可跳过） → 玩家主动点击"揭开迷雾"按钮 → 路径出现 → 选择。仪式感强，玩家掌控节奏。

揭雾做成翻牌动画会很酷，但多一步操作

关于 Agent 之间是否互相讨论

我建议 Agent 各自独立发言，不互相对话。原因：

  • 六位 Agent 各有立场（钱/路/风/心/镜/云），各自从自己维度分析，信息密度更高
  • 互相讨论会增加大量 token 消耗和复杂度，demo 阶段不划算
  • 玩家看到六个不同角度的分析，自己整合判断，更符合"推演"的感觉
