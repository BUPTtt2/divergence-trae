地图拓扑结构 · 三种方向

这是最底层的决策，决定后面所有的迷雾揭开 / Agent 讨论 / 节点交互方式

A · 杀戮尖塔式分支树

从起点向下分叉，每个节点 2-3 条出路。只看得到下一层已揭示节点，再往下的全是迷雾。选了就走，不可回头。

推荐 · 最贴合你说的"路径未知" + "揭开迷雾出现路径"

B · 大富翁环形

闭环路径，骰子决定走几格，节点触发事件。路径是固定的，未知的是骰子和事件，不是路径本身。

和你说的"路径未知"对不上，骰子味太重

C · 网格探索（Roguelike）

2D 网格地图，玩家自由移动，每走一步揭开周围 N 格。路径完全自由，但叙事节奏难控。

自由度太高，多 Agent 讨论的意义会被稀释

关键差异

  • A 分支树：每层选 1 条，决策密度高，Agent 讨论有明确对象
  • B 环形：路径固定，未知在事件，不在路径
  • C 网格：自由度最高，但叙事松散，不适合"讨论后再揭开"的节奏
