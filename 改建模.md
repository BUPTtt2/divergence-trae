正方形悬浮方块 - 有厚度 - 圆角或倒角边缘 - 半透明磨砂材质 - 轻微发光边缘 - 悬浮在虚空中  颜色：  - 奶白 - 浅灰蓝 - 淡紫  不要太暗，不要太灰。  ---  ## 四、用户角色：极简小人  用户角色保持极简。  造型：  - 白色或浅色几何小人 - 圆形头部 - 圆锥身体 - 无五官 - 剪影感 - 比例优雅  不要做复杂人物模型。  动画：  - 轻微悬浮 - 缓慢呼吸 - 转向平滑 - 不要夸张动作  ---  ## 五、智能光球：核心判断者  用户旁边要有一个光球。  ### 光球造型  - 球形 - 半透明 - 内部柔和发光 - 外层有光晕 - 漂浮在用户侧上方  颜色：  - 白色 - 淡金 - 淡紫 - 淡蓝  ### 光球作用  它是系统的核心判断者。  负责：  1. 理解用户输入的决策问题 2. 判断需要哪些 Agent 参与 3. 引导用户提问 4. 总结 Agent 观点 5. 生成最终选择方块 6. 判断是否到达最终决策点  ### 光球动画  - 思考时：缓慢旋转 - 理解中：光晕脉动 - 总结中：粒子环绕 - 完成后：光线变亮  ---  ## 六、用户输入问题后的表现  用户输入决策问题后，不要直接跳对话。  应该有一个轻量的可视化反馈。  ### 1. 问题气泡  用户头顶浮出一个半透明气泡。  里面显示用户输入的问题。  样式：  - 圆角矩形 - 磨砂玻璃质感 - 浅色背景 - 清晰文字 - 从上方淡入 - 轻微漂浮  例如：  \`\`\` 要不要接这个新 Offer？ \`\`\`  ### 2. 光球开始分析  光球靠近用户。  周围出现少量粒子。  光球文字或 UI 提示：  \`\`\` 正在理解你的问题... \`\`\`  ### 3. 光球判断需要哪些 Agent  不是所有问题都需要所有 Agent。  例如：  - 职业问题：钱谷、路向、风眼、心禾 - 感情问题：心禾、风眼、镜我 - 学业问题：路向、心禾、行探 - 投资问题：钱谷、风眼、行探  所以系统需要支持：  \`\`\` 根据问题类型动态加载 Agent \`\`\`  不要定死 6 个 Agent 全部出现。  ---  ## 七、Agent 表现：虚影，不是塔楼  这是最重要的改动。  Agent 不要做成塔楼，不要做成建筑，不要做成复杂角色。  所有 Agent 都做成：  \`\`\` 半透明虚影方块 / 虚影符号 / 虚影头像 \`\`\`  ### Agent 基础形态  每个 Agent 是一个悬浮的半透明方块。  结构：  - 半透明方形底板 - 上方有 Agent 图标 - 旁边显示名字 - 底部有对应颜色光晕 - 整体像一个灵体、投影、虚灵  例如：  - 钱谷：金色虚影方块 - 路向：绿色虚影方块 - 风眼：红灰虚影方块 - 心禾：紫色虚影方块 - 镜我：蓝紫虚影方块 - 行探：青绿色虚影方块  ### Agent 不要做  - 不要塔楼 - 不要建筑 - 不要复杂人物 - 不要厚重模型 - 不要大量装饰 - 不要高面数模型  ---  ## 八、Agent 出现方式：八卦式环绕  用户提问后，光球开始召唤 Agent。  Agent 虚影从虚空中浮现，并围绕用户形成一个八卦式环绕。  ### 布局  用户站在中心。  光球在旁边。  Agent 虚影环绕在周围。  例如：  \`\`\`       钱谷    路向   风眼       用户    心禾   镜我       行探 \`\`\`  但不要太规则，要有自然错落。  ### 出现顺序  不要一次性全部出现。  顺序：  1. 先出现 1 个主 Agent 2. 再出现第 2 个 3. 再出现第 3 个 4. 最后形成环绕  每个 Agent 从虚空中：  - 淡入 - 旋转 15° - 上浮到目标位置 - 光晕亮起  ---  ## 九、Agent 对话交互  Agent 不是一次性说完就消失。  每个 Agent 可以和用户进行轻量对话。  ### 对话表现  Agent 虚影悬浮起来。  旁边出现聊天气泡。  气泡里显示 Agent 的问题或观点。  例如：  \`\`\` 钱谷：先别急着表态。咱们先把数字摊到桌面上。 \`\`\`  然后等待用户回应。  ### 用户回应方式  用户可以通过右侧 UI 输入、选择预设回答、或点击确认。  不要要求复杂自然语言交互，先做轻量版。  ### Agent 小动效  当 Agent 正在思考或说话时：  - 虚影轻微浮动 - 光晕脉动 - 图标轻微跳动 - 气泡逐字显示  不要复杂动画。  ---  ## 十、Agent 对话完成后的表现  一个 Agent 和用户交互完成后，它不要消失。  应该：  1. 虚影回到环绕位置 2. 颜色变稳定 3. 旁边显示一个小勾 4. 光球记录它的观点 5. 下一个 Agent 浮起继续对话  这样用户能看到整个讨论过程。  ---  ## 十一、光球总结阶段  当所有相关 Agent 都对话完成后，光球开始总结。  ### 表现  1. 光球飞到中心上方 2. 所有 Agent 虚影围绕光球旋转一圈 3. 光球周围出现粒子环流 4. 右侧 UI 显示总结内容 5. 用户脚下出现光效八卦图案  ### 脚下光效  用户脚下浮现一个发光的八卦式图案。  它不是 UI，而是 3D 场景中的轻量视觉。  作用：  - 表示正在综合判断 - 表示多个 Agent 观点被整合 - 表示即将生成选择  样式：  - 八卦形 - 半透明 - 发光线条 - 缓慢旋转 - 不要太玄学，要科技感和决策感  ---  ## 十二、生成选择方块  总结完成后，光球生成几个选择方块。  ### 选择方块数量  根据问题类型动态生成。  例如：  - Offer 问题：3 个选择 - 感情问题：2-3 个选择 - 学业问题：3 个选择  不要固定数量。  ### 选择方块样式  每个选择是一个悬浮方块。  结构：  - 半透明方块 - 对应颜色 - 顶部图标 - 标题文字 - 底部光晕 - 可点击  例如：  \`\`\` 机会型选择：金色方块 风险型选择：红灰方块 稳守型选择：蓝灰方块 探索型选择：紫色方块 \`\`\`  ### 选择方块出现方式  选择方块从用户脚下依次浮起。  顺序：  1. 先出现第一个 2. 再出现第二个 3. 再出现第三个 4. 围绕用户两侧展开  不要像菜单，要像从虚空中生成的决策路径。  ---  ## 十三、用户点击选择后的路径生成  用户点击某个选择方块后，不要直接跳转。  要生成路径感。  ### 表现  1. 用户脚下光效亮起 2. 选择方块高亮 3. 从初始平台到选择方块之间生成阶梯 4. 阶梯由一块块小方块组成 5. 阶梯逐块升起 6. 用户小人沿阶梯移动过去 7. Agent 虚影退到两侧或淡化  ### 阶梯要求  - 极简方块阶梯 - 有厚度 - 有轻微高度差 - 不要复杂扶手 - 不要复杂桥梁 - 不要装饰过多 - 颜色与选择方块统一  例如：  - 金色选择 → 金色阶梯 - 风险选择 → 红灰阶梯 - 稳守选择 → 蓝灰阶梯  ---  ## 十四、后续分支路径  到达选择方块后，光球继续判断。  如果还没到最终决策点：  1. 新的 Agent 虚影再次出现 2. 继续讨论 3. 继续生成新的选择方块 4. 用户继续选择路径  如果已经到达最终决策点：  1. 光球发出稳定光 2. Agent 虚影形成环绕确认 3. 出现最终命运卡 4. 显示结果与建议  ---  ## 十五、动态 Agent 配置系统  必须支持动态配置 Agent。  不要写死全部 6 个 Agent。  ### Agent 配置数据结构  每个 Agent 配置：  \`\`\`json {   "id": "qiangu",   "name": "钱谷",   "role": "财务分析师",   "color": "#E8D88A",   "icon": "dollar",   "description": "先看数字，再谈感觉。" } \`\`\`  ### 问题类型配置  例如：  \`\`\`json {   "career": \["qiangu", "luxiang", "fengyan", "xinhe"],   "relationship": \["xinhe", "fengyan", "jingwo"],   "study": \["luxiang", "xinhe", "xingtan"],   "investment": \["qiangu", "fengyan", "xingtan"] } \`\`\`  ### 默认 Agent  可以保留几个核心默认 Agent：  - 钱谷：财务与收益 - 路向：路径与规划 - 风眼：风险与变化 - 心禾：情绪与内在 - 镜我：自我洞察 - 行探：行业与外部信息  但界面上要表现为：  \`\`\` 根据你的问题，已召唤 4 位顾问 \`\`\`  而不是全部硬塞出来。  ---  ## 十六、性能要求  这个项目是 Web App，必须轻。  ### 禁止  - 禁止大量复杂模型 - 禁止高面数建筑 - 禁止每个 Agent 都做复杂建模 - 禁止厚重场景 - 禁止大量粒子堆叠 - 禁止复杂物理 - 禁止长时间加载  ### 推荐方案  - 所有 Agent 用半透明方块 + 图标 - 所有路径用方块阵列 - 所有光效用 shader 或 sprite - 所有文字用 DOM / 3D 文字贴图 - 场景保持低多边形 - 粒子数量控制在 100-300 个 - 保证普通笔记本流畅运行 - 首屏加载快  ---  ## 十七、技术实现建议  继续使用：  - React - React Three Fiber - drei - Framer Motion - Tailwind - localStorage  但 3D 层要极度克制。  ### 3D 层只负责  - 初始平台 - 用户小人 - 光球 - Agent 虚影 - 选择方块 - 方块阶梯 - 背景空间 - 轻量粒子  ### UI 层负责  - 用户输入 - Agent 对话 - 总结内容 - 选择说明 - 决策日志 - 卡牌展示 - 页面导航  不要把所有逻辑都塞进 3D。  ---  ## 十八、动效统一规则  所有动效都要：  - 轻 - 快 - 柔 - 不夸张 - 不笨重  ### 推荐参数  - 浮现动画：0.4-0.8s - 移动动画：0.6-1.0s - 光晕脉动：缓慢循环 - 粒子速度：慢 - 旋转速度：慢 - 缓动：easeOut  不要：  - 弹跳 - 震动 - 爆炸 - 大位移 - 夸张缩放 - 复杂物理碰撞  ---  ## 十九、页面结构保留  保留现有页面结构：  - 首页 - 推演页 - 卡牌册 - 社区 - 设置  但 3D 场景只在推演页作为核心交互区。  首页可以展示一个极简版本的场景预览。  ---  ## 二十、最终验收标准  这版必须满足以下所有条件：  1. 没有复杂塔楼建筑 2. 没有厚重建模 3. 没有一整条丑陋直路 4. 用户初始只站在一个方块平台上 5. 智能光球悬浮在旁边 6. 用户问题能以美观气泡形式出现 7.
