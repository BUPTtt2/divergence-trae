# 演策 v2.0 设计文档

> 目标：把「肉鸽卡牌决策沙盘」升级为「大富翁式 3D 路径探索游戏」，用户一看就想玩，停不下来。

---

## 一、六大 Agent 系统

现有 4 个 Agent（钱谷、路向、风眼、心禾），新增 2 个：

### 新增 Agent

| Agent | 名称 | 定位 | 立场 | 颜色 | 标签 |
|-------|------|------|------|------|------|
| 镜渊 | Mirror | 反思者 | 反思 | #7C5CBF 紫 | 专门反思你过去的决定模式，问"上次类似情况你怎么选的？结果如何？" |
| 云图 | CloudMap | 全局观 | 宏观 | #4A90D9 蓝 | 从宏观视角看 3-5 年趋势，行业周期，经济大环境 |

### Agent 在决策中的角色

每个节点辩论触发时，从 6 个 Agent 中选取 3-4 个参与（不是每次全部 6 个都发言，避免信息过载）：
- 起点（input）：全部 6 个 Agent 发言（首次印象，全面分析）
- 迷雾节点（fog）：选 3 个最相关的 Agent
- 十字路口（crossroad）：选 4 个 Agent（关键决策点）
- 深入节点（deep）：选 2 个 Agent（聚焦深度）
- 命运节点（fate）：不触发辩论

### 首页展示

首页「你的六位智囊」Section 扩展为 6 列（2x3 或 3x2 grid），每个 Agent 卡片包含：
- 头像（首字母 + 对应颜色）
- 名称 + 立场标签
- 一句话描述
- 底部标注其参与阶段（起点/迷雾/十字/深入）

## 二、大富翁式 3D 地图系统

### 核心视觉改动

**从「肉鸽节点地图」改为「大富翁式路径棋盘」**：

1. **俯视视角的 3D 棋盘**：整个沙盘是一个等距投影（isometric）的游戏棋盘，像大富翁棋盘那样有立体感
2. **路径是弯曲的棋盘道**：节点之间用实线连接，像大富翁的格子道
3. **每个节点是一个「地格」**：正方形或六边形的立体地格，有深度感（伪 3D）

### 小人系统

1. **用户创建自己的小人**：
   - 推演开始时，用户可以选择小人外观（4-6 种预设，如：工程师、探险家、忍者、猫咪、机器人、外星人）
   - 小人是简单的像素风/低多边形 3D 形象
   - 使用 CSS 3D transform 或简单的 SVG/Canvas 绘制

2. **小人在地图上移动**：
   - 小人站在当前节点上
   - 移动时有步行动画（逐格移动）
   - 每次掷骰子后，小人按步数沿路径移动到目标节点
   - 到达节点后播放「到达」动画（弹跳 + 粒子）

3. **路径装饰物**：
   - 已走过路径：绿色实线 + 流光效果
   - 未走路径：灰色虚线
   - 路径上有小装饰（金币、问号、星星等像素图标，经过时触发小动画）

### 地图布局

不再是左到右的分层结构，改为**蛇形路径棋盘**：

```
起点 ── A ── B ── C ── D
                        │
F ── E ── [骰子] ── G ── H
│
I ── J ── K ── L ── M
                        │
P ── O ── N ── [骰子] ── 终点
```

- 路径蜿蜒蛇形，每行 5 格
- 拐弯处有弧形连接线
- 起点和终点有特殊造型
- 骰子标记在关键分叉处

### 装饰元素

- **背景**：淡绿色草地纹理 + 微妙网格
- **路径两侧**：随机分布的装饰（小树、石头、金币、问号框）
- **节点类型视觉**：
  - 起点：绿色旗帜
  - 机会节点：金色星星 + 发光
  - 风险节点：红色闪电
  - 中性节点：蓝色箭头
  - 骰子节点：金色骰子图标
  - 终点：紫色奖杯 + 光晕

## 三、骰子升级

### 3D 骰子

- 使用 CSS 3D transform 制作立体骰子
- 掷骰子时有**摇晃 → 旋转 → 弹出 → 定格**动画
- 骰子面上显示 1-6 点数（真实骰子点数图案）
- 定格后有**震动 + 光晕扩散**效果
- 结果横幅：绿色 = 运气好，红色 = 运气差

### 骰子在地图上的位置

- 地图上预设 2-3 个骰子节点（特定位置）
- 到达骰子节点后，骰子动画在地图中央弹出
- 用户点击掷骰子，步数决定前进方向

## 四、可复玩路径系统

### 核心机制

1. **保存进度**：每次推演完成后，将当前路径保存到 localStorage
2. **回退分支**：在推演过程中或结束后，用户可以点击已走过的节点，选择「从这里换一条路」
3. **路径总览**：推演完成后显示「路径地图总览」，标注所有走过的和未走的分支
4. **多次游玩**：
   - 从同一个起点开始，选择不同的分支
   - 地图上用不同颜色标注不同游玩的路径
   - 每条路径都独立记录 Agent 讨论和命运卡

### 回退 UI

- 点击已走过的节点，弹出「路径选择器」面板
- 面板显示：「从这里可以走以下分支」
- 选择新分支后，该节点之后的路径被清除，新的辩论和骰子触发
- 之前的 Agent 讨论记录保留在「历史对话」中

## 六、技术实现方案

### 不受限的技术选择

- **地图渲染**：Canvas 2D（性能好 + 足够表现伪 3D）或 Three.js（真 3D 但重）
  - 推荐：Canvas 2D + CSS transform 组合，够用且轻量
  - 等距投影用 Canvas 的 `setTransform` 实现
- **小人动画**：CSS sprite sheet 或 Canvas 绘制 + `requestAnimationFrame`
- **骰子**：CSS 3D cube（`transform: rotateX() rotateY()`）+ `preserve-3d`
- **路径系统**：数据结构从「树」改为「有向图」，每个节点存储 `edges: [{target, label, type}]`
- **进度存储**：`localStorage` 存储完整的游玩历史
- **部署**：Vite build → 静态文件 → 任意静态托管（Vercel / Netlify / GitHub Pages）
